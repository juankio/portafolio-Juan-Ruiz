## 🌟 Sobre el Proyecto

Este es el repositorio de mi portafolio web personal. El objetivo no es solo mostrar mis proyectos, sino **demostrar mis habilidades** directamente en el código y en la interfaz.

El diseño está enfocado en una **excelente experiencia de usuario (UX)**, micro-interacciones, animaciones fluidas y soporte completo para i18n (Internacionalización). Todo potenciado por el rendimiento de **Nuxt 3**.

### ✨ Características Principales

- ⚡️ **Rendimiento Extremo**: SSR, optimización de imágenes y bundle ligero.
- 🎨 **Diseño UI/UX Premium**: Inspirado en un estilo neo-brutalista, minimalista y con acentos estilo grafiti.
- 🌗 **Modo Oscuro/Claro**: Soporte nativo para temas con transiciones fluidas.
- 🌍 **Multilenguaje (i18n)**: Textos disponibles en Español (ES) e Inglés (EN).
- 📱 **100% Responsivo**: Mobile-first y adaptado a cualquier pantalla.
- 🤖 **Potenciado por IA**: Desarrollo y optimización usando agentes de IA (Claude, Opencode, Open claw) para flujos 10x más rápidos.
- 📦 **API de GitHub Dinámica**: Muestra los repositorios dinámicamente conectando a GitHub.

## 🚀 Instalación y Uso local

Sigue estos pasos para correr el portafolio en tu máquina local. Estoy usando `bun`, pero puedes usar `npm` o `yarn` o `pnpm`.

### 1. Clonar el repositorio
```bash
git clone https://github.com/juankio/portafolio-Juan-Ruiz.git
cd portafolio-Juan-Ruiz
```

### 2. Instalar dependencias
```bash
bun install
```

### 3. Configurar variables de entorno (Opcional)
Crea un archivo `.env` en la raíz (puedes basarte en un `.env.example` si existe) y añade tu token de GitHub si necesitas más límite de API:
```env
GITHUB_TOKEN=tu_token_personal_aqui
```

### 4. Modo Desarrollo
```bash
bun run dev
```
> El proyecto estará corriendo en `http://localhost:3000`

### 5. Compilar para Producción
```bash
bun run build
# Para previsualizar:
bun run preview
```
